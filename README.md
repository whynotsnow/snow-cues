# Snow Cues v1.0

Snow Cues v1.0 是一个纯浏览器运行的安全本地优先密码系统。系统基于 `master_password` 与单条密码的 `entrySecret`（关键密钥，旧名 `runtime_salt`）派生密码，只在本地加密保存派生后的输出，不保存关键密钥或可重建关键密钥的材料。

## 技术栈

Vite + React 19 + TypeScript，纯手写 CSS，无 UI 组件库。构建产物为纯静态站点。

测试使用 Vitest + Testing Library + jsdom + fake-indexeddb。

浏览器依赖 WebCrypto API（PBKDF2、HMAC-SHA256、AES-GCM），所有主流现代浏览器均内置支持。

## 常用命令

```bash
npm install
npm run typecheck
npm run test
npm run build
```

## 项目结构

| 目录 / 文件 | 职责 |
|---|---|
| `src/session-manager/` | 主密码到 WebCrypto key 的派生、会话超时与清除 |
| `src/rule-registry/` | 静态纯函数规则注册表，内置 `v1-hmac`、`v2-pbkdf2` 及预留 `v3-argon2` |
| `src/crypto-engine/` | 规则执行、输出编码（base62/base64/自定义字符集）、密码输出适配、AES-GCM 加解密 |
| `src/recovery-aid/` | 关键密钥记忆提示的专用加密与解密 |
| `src/storage-engine/` | IndexedDB 读写、字段白名单、空间 profile、密码组、空间关系和迁移批次持久化 |
| `src/space/` | 空间 policy、诊断、克隆、导入导出和迁移编排 |
| `src/lib/bytes.ts` | 字节与编码转换工具（utf8、base64、concat） |
| `src/App.tsx` | React 应用入口，只装配 UI controller 与视图 |
| `src/ui/` | UI 层组件、页面级 controller hook、表单、条目运行时 hook 与 UI 能力聚合 |
| `src/ui/pages/` | hash 路由承载的页面入口，只负责组合 controller 状态和展示组件 |
| `src/ui/components/` | 三栏工作台、全局操作指引、空间主页卡片、密码管理、规则管理、条目卡片等纯展示组件 |
| `src/ui/hooks/` | 空间访问、规则 profile、密码条目动作、表单输入和测试清理等 UI 逻辑 hook |
| `src/styles.css` | 手写样式，无第三方 UI 依赖 |

## 安全模型

- `master_password` 永不持久化。
- `entrySecret` / 关键密钥永不持久化；`runtime_salt` 是历史命名，也不得写入存储。
- 会话中的 `CryptoKey` 均为 non-extractable，仅保存在内存中。
- 会话有 5 分钟空闲超时和 30 分钟绝对超时，过期后自动清除所有敏感状态。
- 条目加密使用关键密钥 + 会话 key 临时派生的 AES-GCM key，不是直接使用会话固定 key。
- 应用使用“本地存储空间”选择本地数据分区，不提供后端账号系统；`spaceId` 是非敏感分区标识。
- 新空间、空空间或克隆/导入后的目标空间在没有可校验密码条目时，会在空间主页先提示设置空间主密码。该操作只建立当前浏览器内存 session，不持久化 `master_password` 或校验材料；本次 session 有效期内后续初始化、创建、迁移等操作不再重复要求输入主密码。
- 进入空间不会因为校验失败被阻止；如果当前空间已有密码，进入后会默认选择最近更新的一条已存储密码作为校验条目，并在空间主页自动展开校验输入。用户也可以在空间主页改选另一条已知关键密钥的密码完成校验。系统不保存密码校验材料。
- 当前空间已有密码且校验未完成时，只能查看列表状态，并在空间主页查看待校验条目的提示或解密完成校验；不能创建、修改、废弃、变更规则或执行测试清理操作，避免错误的 `master_password` 生成新的密码体系。
- 通知按系统级、页面级、区域级和操作反馈分层展示。系统级通知优先使用浏览器 Notification API，用户拒绝或浏览器不可用时降级为站内组件；系统通知文案不得包含 `spaceId`、平台、备注、关键密钥、密码或记忆提示等隐私信息。页面级通知统一展示在中间主内容的工作区路由页面顶部，并可携带受控页面跳转动作。页面级通知只说明当前页面或空间状态、风险、限制和门禁原因，不承担普通操作指导；历史空间和归档空间的顶部通知尤其不能写成“可以做什么”的操作建议。
- 右侧全局“用户操作指引”以多个动态卡片展示当前下一步、可用操作、相关流程或受阻流程，负责回答“现在能做什么 / 如果要继续需要先做什么”。它可根据流程优先级展示主卡和次级卡，已完成或不存在的流程不生成卡片；指引不持久化已读状态，不保存任何敏感信息。
- 单条密码永不保存 `ruleId`、scene/context 或其他可重建派生输入的元数据。
- 空间外的“游离密码”只在当前页面内存中临时预览；派生密钥、预览结果和生成历史不会写入 IndexedDB、localStorage、sessionStorage、URL 或迁移队列。
- IndexedDB 的密码条目只保存 AES-GCM 加密后的核心密码输出、可选的加密记忆提示 `encrypted_memory_hint`，以及平台、密码组归属、普通备注等非敏感识别字段。
- 密码组用于管理同一平台或系统下的多个账号，保存明文组名、普通说明和声明式输出适配策略；这些字段是隐私元数据，不是密码学秘密，不参与核心密码生成或解密。
- 密码输出适配只发生在核心密码解密之后。系统用公开、确定性的内置算法按常见策略或自定义参数生成临时适配密码，不覆盖已保存的核心密码密文。
- `description` 是明文普通备注，不是关键密钥提示；关键密钥记忆提示只能以 `encrypted_memory_hint` 密文保存，不参与密码生成、密码解密或自动恢复。
- 创建密码时可以填写“关键密钥记忆提示，可选”；已存储密码列表默认不展示提示输入框，可点击“查看记忆提示 / 隐藏记忆提示”作为解密辅助。
- 空间校验完成后，可点击“编辑条目”维护平台、普通备注和记忆提示；编辑态中记忆提示默认隐藏，用户需要明确点击“显示提示”只读查看，或点击“解锁编辑 / 添加提示”后才可修改。
- 编辑提示会重新加密写入 `encrypted_memory_hint`；清除提示必须点击明确的“清除提示”，避免把空输入误当作删除。保存或取消编辑后，记忆提示明文草稿会从内存状态清除。
- 查看记忆提示只依赖当前空间会话，不需要再次输入关键密钥；如果查看提示后仍无法解密，界面会建议用户重置平台密码并将当前条目标记为废弃，不会自动废弃。
- 空间级 profile 会保存初始化后的 `ruleChain` 和参与规则链的声明式导入规则 manifest，用于后续进入同一空间时恢复同一套全局规则配置。
- 规则系统只使用静态注册的纯函数；不实现运行时加载 JavaScript 规则。
- 新建密码和解密密码都需要用户输入关键密钥。该关键密钥只参与当前操作中的密码生成和临时 key 派生，不会写入存储。
- 导入规则只接受单个声明式 JSON 或声明式 JSON 数组，并映射到允许的内置算法模板（`hmac-sha256` 或 `pbkdf2-sha256`）；系统不会执行导入的 JavaScript 代码。
- 规则链是全局初始化配置，不属于某一条密码；单条密码不会保存 rule 信息。默认规则链为双规则 `v1-hmac → v2-pbkdf2`。
- 新建密码前需要先初始化规则链；创建时会展示当前生效规则链和处理流程。
- 核心密码输出编码支持三种模式：Base62、Base64、自定义字符集；默认输出长度 24（可调范围 8–64）。
- 输出适配页提供常见密码策略预设和自定义策略。内置预设不宣称兼容具体平台，选择内置预设时参数锁定；切换到“自定义当前策略”后可基于当前参数调整。
- 新建密码成功后只提示已加密保存，不在新建表单底部常驻展示密码输出。
- 当前版本允许修改平台和描述，暂不支持修改密码内容；编辑条目时密码位置显示“当前不支持编辑密码”以保持布局稳定，并禁用删除密码。
- 密码可以标记为废弃；废弃不会删除数据，废弃后平台不可修改，描述仍可修改。
- 创建和解密过程带有 loading 状态；解密失败会提示检查关键密钥。

## 空间、导入导出与迁移

- `spaceId` 是本地数据分区标识，不支持修改；空间主密码也不做原地修改。需要更换空间主密码时，应创建新空间并迁移条目。
- 空间持久状态为 `active`、`deprecated`、`archived`，界面分别显示“正常 / 历史 / 归档”。`deprecated` 空间冻结创建、编辑、废弃、规则变更、迁移管理和测试清理，但保留历史查看和必要解密；若已有密码且校验未完成，右侧指引应称为“准备解密历史密码”，而不是复用 active 空间的写入门禁式“空间校验”。`archived` 空间用于封存，不支持密码解密或日常派生，但可在会话能力允许时查看已保存的记忆提示。
- 空间外页面展示本地空间索引和创建空间入口。空间索引只展示本地空间元数据，不展示密码条目。
- 空间外主菜单包含“空间工作台”和“游离密码”两个平级页面。“空间工作台”展示本地空间索引和创建空间入口；“游离密码”页面可用派生密钥和默认内置规则链临时预览密码，并由用户决定预览时是否应用声明式输出策略。游离密码迁入空间时只迁入派生密钥草稿，进入目标空间后按该空间稳定 `ruleChain` 正式生成并加密保存；用户仍需要填写平台、密码组、普通备注和可选记忆提示。
- 空白创建会先进入新的临时空间；只有初始化规则链或创建密码后，才会持久保存空间记录。克隆或导入会创建目标空间、profile 或迁移队列，因此会写入本地空间索引。
- 空间外创建入口点击后先选择创建方式，再按方式展示字段：空白创建、从已有空间 clone 配置、从已有空间 clone 配置 + 密码迁移队列、从导入文件创建空间、从导入文件创建空间 + 密码迁移队列。
- 应用使用后台式三栏工作台：左侧是品牌、空间范围或当前空间信息和主菜单，中间是当前页面主内容，右侧是全局用户操作指引。左侧不直接展示内部 `UiState` 状态机标签；空间外展示本地空间工作台范围，空间内展示当前 `spaceId` 和空间持久状态。桌面端保持三栏，移动端上下堆叠，菜单变为可换行按钮组。进入空间后默认进入空间主页；工作区使用 hash 路由承载“空间主页 / 规则管理 / 输出适配 / 密码管理”，主导航顺序保持一致。空间主页展示当前空间详情、空间关系、空间校验、空间主密码设置、迁移情况和当前空间操作；若当前空间已有密码，空间校验区域会默认展开最近更新条目的校验输入，并允许用户改选其他条目完成校验。若当前空间没有可校验密码且本次 session 尚未建立，空间主页会先展示“设置空间主密码”入口。迁移情况位于当前空间操作之前，方便目标空间先处理迁移队列。密码管理列表仍保留完整条目列表，但校验入口收敛到空间主页。
- 当前空间操作只针对已进入空间：可 clone 当前空间配置到新的空间、从当前空间 clone 配置 + 密码迁移队列、导出当前空间配置 JSON、导出当前空间完整备份 JSON。clone 当前空间完成后会直接进入目标空间主页。
- 仅克隆配置只复制 Space 元数据和空间 profile，不复制密码条目，不创建空间指向关系，也不流转旧空间状态。
- 克隆或导入密码条目时，旧条目不会直接写入目标空间的正式 `password_entries`，而是写入独立的 `migration_batches` 和 `migration_entries` 迁移队列。
- 从已有空间 clone 配置 + 密码迁移队列时，目标空间不会立刻保存正式 `space_profile`。来源规则链会作为迁移草稿载入规则管理页，用户需要先在目标空间手动确认初始化规则链；规则链确认后，系统会自动把迁移批次从 `draft` 置为 `ready`，用户下一步直接校验来源空间。
- 游离密码的“迁移到空间”是用户交互语义，不使用 `migration_batches` 或 `migration_entries`，不创建空间关系，也不流转任何来源空间状态。
- 待迁移条目保存的是旧空间密文、平台、备注和来源状态；它不是 `PasswordEntry`，不能出现在正式密码列表中。
- 迁移支持两种模式：保持平台密码不变（解旧密文后用新空间重新加密）和按目标空间规则重新生成（用户确认外部平台已更新后才落盘）。
- 迁移期间同时存在来源空间 session 和目标空间 session。目标空间 session 可先在空间主页通过“设置空间主密码”建立；右侧全局指引会在存在未完成迁移批次时按“设置目标空间主密码 → 初始化目标规则链 → 校验来源空间 → 逐条迁移或跳过 → 完成来源空间流转”的顺序提示当前迁移步骤。若迁移批次已完成且已存在 `successor_of` 接替关系，右侧不再展示迁移流程指引。来源空间校验复用“选择一条已有条目 + 旧空间主密码 + 旧关键密钥”的校验方式，校验完成后来源校验卡片只保留已校验提示。校验中、校验失败和校验成功都会在卡片内展示通知，失败或成功也会进入系统通知层。
- 旧空间主密码只输入一次并仅保存在内存中；每条迁移仍需输入旧 `entrySecret`，新 `entrySecret` 可由用户选择复用旧值或重新输入。若迁移按钮禁用，条目卡片会显示需要先初始化目标规则链、等待迁移批次自动就绪、完成目标空间会话或完成来源空间校验等具体原因。
- 迁移批次默认勾选“迁移完成后自动流转来源空间状态”。若用户在迁移前关闭该选项，全部条目迁移或跳过后不会自动创建 `successor_of` 关系，也不会自动把来源空间标记为 `deprecated`；用户需要在空间主页手动点击“手动流转来源空间状态”。关系只用于展示来源和接替，不跨空间继承密钥或密码条目。
- 导出配置 JSON 只包含空间、profile 和关系；导出完整备份会额外包含旧空间加密密码条目。导出文件不得包含 `master_password`、`entrySecret`、明文密码或校验材料。
- 全局“测试数据工具”始终展示在页面底部，和是否进入空间无关；可用于清空全部本地数据、删除指定空间，以及在已进入空间且策略允许时清空当前空间密码数据或重置当前空间规则链配置。

## IndexedDB 维护经验

- IndexedDB schema 变更必须提升 `DB_VERSION`，并在 `onupgradeneeded` 中补齐所有必需对象仓库和索引。
- 浏览器开发过程中可能出现“数据库版本号已升级，但对象仓库不完整”的状态，尤其是 Chrome 在旧连接关闭、页面并发打开和开发热更新交错时更容易暴露。
- 不能只在 UI hook 中捕获 `transaction` 异常并展示错误；这会把 schema 损坏从阻塞流程变成页面错误，并没有修复根因。
- `openDatabase()` 会在打开成功后校验必需对象仓库。如果发现同版本 schema 不完整，会关闭旧连接并使用更高内部版本重新打开数据库，触发升级流程补齐缺失仓库。
- 新增对象仓库时需要补低层测试，模拟“当前版本数据库缺少新仓库”的坏状态，确认应用能自动修复而不是要求用户手动清空浏览器数据。

## UI 层结构

UI 层位于 `src/ui/`，以 hook 作为状态和业务逻辑的边界，组件尽量只负责渲染和派发事件。

- `useAppController` 是顶层组合 hook，负责汇总空间、规则、条目和页面动作并暴露给视图。
- `useSpaceAccessController` 负责进入/离开空间、空间检测、刷新条目、会话存活检查和 `withLiveSession`。
- `useRuleProfileController` 负责规则链草稿、导入规则、profile 恢复和规则链初始化。
- `usePasswordEntryController` 负责新建密码、解密、记忆提示、条目编辑和废弃。
- `useSpaceIndexController` 负责空间外本地空间索引、空间外创建空间和创建后进入空间主页。
- `useSpaceManagementController` 负责空间主页中的当前空间操作、导入导出、迁移批次、来源空间内存 session 和单条迁移。
- `useWorkspaceActions` 负责全局测试数据工具、当前空间测试清理和新建密码入口跳转。
- `useCreatePasswordForm`、`useEntryRuntimeState` 保存局部 UI 状态；这些状态不写入存储。
- `entryCapabilities.ts` 聚合条目级 UI 能力（解密、查看提示、编辑、废弃和禁用原因），组件优先消费聚合能力而不是分散拼接底层 policy。
- `AppView` 负责三栏工作台、hash 路由、左侧主菜单和右侧全局操作指引的装配；`pages/` 下放置路由页面入口，`components/` 下放置可复用展示组件和空间主页卡片，复用现有 `src/styles.css`，不引入 UI 组件库。
- `guidance.ts` 集中根据 controller 状态计算 `UserGuidance` 多卡片队列；`GuidancePanel` 只渲染指引，不直接依赖业务 hook。新增流程指引时优先新增独立 builder，并明确卡片状态是下一步、可用操作、相关流程还是受阻流程。

## 部署

`npm run build` 会生成静态 `dist/` 文件夹。Vite 配置已设置 `base: "./"` 使用相对路径，可直接部署到 GitHub Pages 或任何静态托管，也可交给 Syncthing 等外部文件同步工具同步。

Syncthing 不属于应用逻辑，不要在应用内实现同步功能。
